## 案件概要
### 背景・課題
テニス部は岡山・広島・山口・鳥取・島根の中国地方5県で構成される広域組織。各県ごとに活動費が発生し、加えてテニス部全体としての活動費も発生する。現状は予算の利用状況を一元的に把握する仕組みがなく、

- 県ごとの活動費がリアルタイムで集計できない
- 領収証・インボイス情報の保管が散逸しがち（紙・個別端末に分散）
- 県活動費として充当するか全体活動費として処理するかの承認フローが属人化
- 予算と執行実績の差分が見えず、ガバナンスと説明責任が不十分

という課題を抱えている。

### 目的・ゴール
- 5県＋全体の活動費・予算をWeb上で一元管理する
- 一般ユーザー（部員）が領収証画像/PDFとインボイス情報を添えて活動費を簡単に登録できる
- 県管理者が承認・充当判断をスムーズに行える
- 全体管理者が県横断で予算執行状況を可視化できる
- インボイス制度（適格請求書）に対応した記録保管を実現する
- **最高レベルのUX × あっと驚くおしゃれなUI**で、「使うのが楽しい」予算管理ツールにする

### ターゲットユーザー
1. **全体管理者**（テニス部本部）: 全体予算策定・全体活動費登録・5県横断レポート参照
2. **県管理者**（各県幹事 × 5名想定）: 自県の活動費承認・県活動費充当判断・自県予算管理
3. **一般ユーザー**（部員）: 自分が立て替えた活動費の登録申請・領収証アップロード

## 要件概要
### 必須要件（Must）
1. **ユーザー種別と権限分離**: 全体管理者 / 県管理者（県スコープ）/ 一般ユーザー（自分のレコードのみ）の3階層 RBAC + 県スコープの RLS
2. **活動費入力（一般ユーザー）**: 日付 / 内容 / 金額 / 領収証有無 / 領収証画像・PDF添付 / インボイス登録番号 / 所属県の自動付与
3. **承認・充当フロー（県管理者）**: 申請一覧 → 承認/差戻 → 県活動費 or 個人負担 or 全体活動費候補としての分類
4. **県別集計・予算管理**: 各県の予算枠、執行額、残額、執行率の可視化
5. **全体管理（全体管理者）**: 5県横断の集計、全体予算と執行状況、全体活動費の登録・管理
6. **領収証ストレージ**: Supabase Storage + 添付ファイル即時プレビュー（画像/PDF）
7. **インボイス対応**: 適格請求書発行事業者登録番号（T+13桁）の保存・検索
8. **認証**: Supabase Auth（メール + パスワード or マジックリンク）
9. **絵文字禁止 / Heroicons 使用 / クリーンなクライアント企業向けデザイン**（組織標準）

### 希望要件（Should）
1. CSV / Excel エクスポート（会計ソフト連携・年度報告用）
2. 月次・年次レポート画面（県別グラフ + 全体グラフ）
3. 通知機能（申請が来た / 承認された / 差戻された のメール通知）
4. 監査ログ（誰が・いつ・何を・承認/変更したか）
5. ダークモード対応（next-themes）

### あれば嬉しい（Could）
1. OCR / VLM による領収証自動入力（PRJ-002/PRJ-005 のノウハウ流用余地あり）
2. 予算超過アラート（県管理者・全体管理者へPush / メール）
3. モバイル対応（PWA でスマホから領収証撮影 → 即登録）
4. 年度繰越処理（予算/活動費の年度切替）

### 対象外（Won't）
1. 給与計算・経費精算の銀行振込連動（要件外）
2. 5県以外への拡張（中国地方テニス部固有）

## 制約条件
- **技術的制約**: 組織標準（Next.js App Router + TypeScript + shadcn/ui + Tailwind + Supabase + Vercel）を初期案として採用。詳細は `organization/rules/tech-stack.md` 準拠
- **デザイン制約**: AI感を出さないクリーンなデザイン / 絵文字禁止 / Heroicons / 「最高レベルのUX」「あっと驚くおしゃれ」
- **その他制約**:
  - 個人情報（部員氏名・連絡先）と金銭情報を扱うためセキュリティ重視
  - インボイス制度の登録番号フォーマット（T + 半角数字13桁）バリデーション必須
  - 5県スコープの RLS が漏れると別県の経費が見えてしまうため、テナントスコープ設計を Phase 0 で固める
