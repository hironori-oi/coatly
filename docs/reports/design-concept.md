# PRJ-015 デザインコンセプト

- 案件: テニス部予算管理Webアプリ（仮称）
- 作成日: 2026-04-26
- 担当: デザイン部門
- ステータス: Phase 0 / コンセプト提案（CEO 決裁待ち）

## 0. デザインの北極星（Design North Star）

> **「会計の厳格さ」と「コートに立つ瞬間の余白」が同居する、静かで上質な予算管理体験**

オーナーの要望「あっと驚く × おしゃれ × 最高レベル UX」を、安易な装飾やAI感の強いグラデーションで作るのではなく、
- 余白の取り方
- タビュラー数字（金額に揃った数字）
- マイクロインタラクションの上質さ
- 1色だけのアクセントの効かせ方

で実現する。組織標準の「AI感を出さないクリーンなデザイン」を厳守しつつ、Linear / Mercury / Stripe Press の系譜にある "Quiet Confidence" を狙う。

## 1. ムード・トーン設計（3案）

### A案: Quiet Luxury Sport（推奨 ★）
- **コンセプト一行**: モノクロ高級紙の上に、コートラインのような一筋の緑だけを残す。
- **カラーパレット（HEX）**:
  - Ink Black: `#0A0A0B`
  - Paper White: `#FAFAF7`（ほんのり生成りの白／長時間見続けても疲れない）
  - Court Green（アクセント）: `#1F6B4A`（深いコートグリーン／差し色1色）
  - Soft Stone: `#E7E5DF`（surface / card）
  - Border Hairline: `#D8D6CF`
  - Muted Text: `#6B6B6B`
  - Signal Amber: `#B8741A`（warning / 予算超過注意）
- **想定フォント**:
  - 見出し: Inter Display（または Geist Sans の太字グレード）／日本語: Noto Sans JP
  - 本文: Geist Sans / Noto Sans JP
  - 数値（金額・登録番号）: **Geist Mono または Söhne Mono**（タビュラー）
- **想定アニメーション特徴**:
  - 200-280ms の `cubic-bezier(0.2, 0.8, 0.2, 1)` をベース easing に統一
  - KPI カードは "lift" ではなく "ink fade in"（不透明度＋4px 上方向）
  - 数字は count-up（300-600ms、tabular-nums）
- **強み**: 30〜60代の県管理者にも違和感がない。会計データとの相性が抜群。実装コスト低。
- **弱み**: 単体だと「驚き」が弱く見えがち → §4 の装飾要素3点で補う。

### B案: Editorial × Court Vision
- **コンセプト一行**: 雑誌のように大胆なタイポと、コート緑のアクセントで魅せる。
- **カラーパレット（HEX）**:
  - `#111111` / `#FFFFFF` / `#22573A`（Court Green）/ `#F2EDE5`（Cream）/ `#E2DCCF`（Border）/ `#9E9A93`（Muted）/ `#C2410C`（Pop Orange）
- **想定フォント**: 見出し = Fraunces / Tiempos（セリフ）、本文 = Inter、数値 = Söhne Mono
- **想定アニメーション**: ページ遷移時にタイトルが横スライドで入場、グラフは左→右に描画。
- **強み**: 個性が際立つ／ポートフォリオ映え。
- **弱み**: B2B 経費管理アプリとしては"主張が強すぎる"。実用主義の県管理者には敬遠される懸念。

### C案: Glassmorphism Tactile（Apple Vision OS 系）
- **コンセプト一行**: 淡いグラデ背景 × ガラス質カード × 繊細なパララックス。
- **カラーパレット（HEX）**:
  - `#0F1115` / `#F4F6FA` / `#3DBE8B`（Mint Green）/ `#1A2332` / `#2A3548` / `#9CA3AF` / `#FB923C`
- **想定フォント**: SF Pro 系（システム）+ Geist Mono
- **想定アニメーション**: blur-backdrop、tilt、parallax。
- **強み**: 視覚的インパクト最大。
- **弱み**: **デザインガイドライン §1「グラスモーフィズムの乱用を避ける」「AI感を出さない」と直接抵触**。低スペック端末でのパフォーマンス劣化。**採用非推奨**。

### 推奨: **A案（Quiet Luxury Sport）**
理由:
1. 組織ガイドライン（クリーン・AI感NG・アクセント1色）と完全整合。
2. 金銭・承認という「真面目な業務」に対する信頼感が最大化。
3. "驚き"は §4 の装飾要素3点で確保 → 実用性と意外性の両立。

### 2.3 スペーシング（4 / 8 / 12 / 16 / 24 / 32 / 48 / 64）
- カードパディング: 24
- セクション間: 48 (デスクトップ) / 32 (モバイル)
- フォーム要素間: 16
- インライン要素間: 8

### 2.4 角丸 / シャドウ / ボーダー
- 角丸: `--radius` = 10px（shadcn デフォルト）/ Card は 14px / Button は 8px / Pill（Status）は full
- ボーダー: 1px hairline (`--border`)、ホバー時のみ accent に切り替え
- シャドウ:
  - `shadow-xs`: `0 1px 2px rgba(10,10,11,0.04)`（カード default）
  - `shadow-md`: `0 8px 24px -8px rgba(10,10,11,0.12)`（hover / focused card）
  - `shadow-pop`: `0 24px 64px -16px rgba(31,107,74,0.18)`（モーダル・特別な強調）
- ガラス質・大ぼかしは禁止（ガイドライン準拠）

## 5. アクセシビリティ（WCAG 2.2 AA）

### 5.1 コントラスト
- 本文 (16px) / 背景: `#0A0A0B / #FAFAF7` = 19.7:1（AAA）
- Muted Text (`#6B6B6B / #FAFAF7`) = 5.4:1（AA 通過）
- Court Green (`#1F6B4A`) on Paper White = 6.2:1（AA 通過）
- Dark mode: foreground `#ECEAE3 / #0C0E0D` = 16.8:1（AAA）
- すべての主要ペアを Phase 1 で再検証（`@adobe/leonardo` 想定）

### 5.2 フォーカスリング
- すべてのインタラクティブ要素に `--ring` 2px outline + 2px offset
- `:focus-visible` のみ表示（マウスクリック時は非表示）
- ダークモードでも視認できる明るい accent を使用

### 5.3 キーボード操作
- すべての操作（承認・差戻・充当切替）をキーボードで完結可能に
- 承認画面: `j/k` で行移動、`a` で承認、`r` で差戻、`?` でヘルプ
- ダイアログは Esc で閉じる、Tab はトラップ

### 5.4 スクリーンリーダー
- アイコン単体ボタンは `aria-label` 必須（ガイドライン §アイコン に準拠）
- KPI カウントアップは `aria-live="polite"` で完了値のみ通知（途中の数字は読み上げない）
- グラフは `<svg role="img"><title>...</title><desc>...</desc>` + データテーブル併設

### 5.5 領収証画像の代替テキスト
- 自動 alt: `「{日付} / {申請者} / {金額} の領収証画像」`
- ユーザー任意で「メモ」入力時はそれを alt に追加
- PDF は `<object>` ではなく PDF.js で SVG 化 + ダウンロードリンク併記

### 5.6 prefers-reduced-motion
- カウントアップ → 即値、軌跡・パララックス → 静止、ホバー lift → opacity のみ

## 6. アイコン・ビジュアル

### 6.1 Heroicons
- 標準: 24px Outline（ナビ・ツールバー）/ 20px Solid（アクティブ・小サイズ）
- 主要マッピング:
  | 用途 | アイコン |
  |---|---|
  | ホーム | HomeIcon |
  | 承認 | CheckBadgeIcon (solid for confirmed) |
  | 差戻 | ArrowUturnLeftIcon |
  | 領収証添付 | PaperClipIcon |
  | 金額 | BanknotesIcon |
  | 県切替 | MapIcon |
  | レポート | ChartBarIcon |
  | フィルタ | AdjustmentsHorizontalIcon |
  | 設定 | Cog6ToothIcon |
- アイコン単体ボタンは必ず `aria-label`。

### 6.2 イラストレーション方針
- **第三者 SaaS イラスト（unDraw 等）の汎用イラストは原則使用しない**（AI感・量産感が出る）
- 代わりに **1px stroke の単色ラインアート（accent green）を自社で描き起こす**:
  - コート俯瞰図
  - ラケットと机
  - 空のロッカー（空状態）
  - 5県マップのシルエット
- すべて SVG / `currentColor` 利用でテーマ追従

### 6.3 Empty State 方針
レイアウト: **イラスト（120-160px）** + **タイトル h3** + **説明文 body-sm** + **CTA Button (primary)**

例（活動費がまだない一般ユーザー画面）:
```
       [ラケットラインアート 144px]
     まだ活動費の申請がありません
   立て替えた費用を、領収証と一緒に登録しましょう
            [▶ 新規申請を作成]
```

### 6.4 ロゴ・アプリ名
- 仮ロゴ: `TENN` というワードマーク（Inter Display 700） + 1px 緑のアンダーラインがコートラインに変化するサイン
- 正式ロゴは Phase 1 開始時にクライアント承認後に確定

## 7. CEO への推奨事項

### 7.1 推奨ムード案: **A案 — Quiet Luxury Sport**
- 理由: 組織ガイドライン（クリーン・AI感NG・アクセント1色）と完全整合し、金銭・承認業務に必要な信頼感を最大化。"驚き"は装飾要素3点で確保することで実用性と意外性を両立できる。

### 7.2 必ず採用すべき "あっと驚き" 要素 TOP 3
1. **中国地方5県インタラクティブマップ**（全体管理者ダッシュボードの主役）
   - 5県の予算執行率を accent green の濃淡で表現する choropleth。クリックで県別ビューに遷移。
   - SVG だけで実装可、パフォーマンス影響なし、ガイドライン抵触なし。
   - **このプロダクトのシグネチャ・ビジュアル**となる。
2. **タビュラー数字のカウントアップ**（全 KPI カード共通）
   - `tabular-nums slashed-zero` のモノスペース数字が 600ms で実値へ count-up。
   - 「会計の正確さ」と「使うのが楽しい」の両立を、わずか数行のコードで実現。
3. **テニスボール軌跡で予算消化を可視化**
   - 県管理者ダッシュボード上部の細い帯に、コートの全長を予算 100% と見立てたボールが進む。月切替で軌跡が再描画。
   - テニス部という案件文脈に強くフィットし、初見で必ず話題になる演出。

### 7.3 デザイントークン採用可否
本ドキュメント §2 のトークン定義（カラー・タイポ・スペーシング・シャドウ・モーション）を Phase 1 開発の正式仕様として採用することを推奨。shadcn/ui の CSS 変数体系にそのまま接続可能。

### 7.4 次のアクション（CEO 決裁後）
1. デザイントークンを `app/styles/tokens.css` として実装（開発部門）
2. shadcn/ui を案件ベースに導入し、Button / Card / Dialog / Sheet / Form をトークン適用カスタマイズ
3. 中国地方マップ SVG 用に 5県シルエットアセットを制作（デザイン部門）
4. KPI カウントアップ・承認マイクロインタラクションのプロトタイプ実装（Framer Motion）
5. WCAG 2.2 AA 自動チェックを CI に組み込み（`axe-core`）
