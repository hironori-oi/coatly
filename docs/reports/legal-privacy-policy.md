## 1. 取り扱う個人情報

### 1.1 ユーザー登録時に取得

| 項目 | 必須 | 利用目的 | 保存先 |
|------|------|---------|--------|
| メールアドレス | ◯ | 認証 / 通知 / 招待 | `users.email`（Turso） |
| 氏名（フルネーム） | ◯ | 申請者表示 / 承認画面 | `users.full_name`（Turso） |
| 所属県 / グループ | ◯ | スコープ判定 / 承認ルーティング | `users.home_group_id`（Turso） |
| パスワード | ◯ | ログイン認証 | bcrypt ハッシュ（Better Auth） |

### 1.2 申請時に取得

| 項目 | 取得方法 | 利用目的 | 保存先 |
|------|---------|---------|--------|
| 領収証画像 / PDF | アップロード | 経費証憑保管 / 承認時の確認 | Vercel Blob or R2（米国） |
| 領収証に映り込む情報（購入店舗・金額・日付・購入者氏名・支払方法・カード末尾4桁等） | 受動取得 | 経費証憑保管 | Vercel Blob or R2 |
| 申請日・支出日・金額・内容・インボイス番号 | 入力 | 経費精算 / 集計 / 監査 | `expenses` テーブル（Turso） |

### 1.3 自動取得

| 項目 | 取得方法 | 利用目的 | 保存先 |
|------|---------|---------|--------|
| IP アドレス | リクエスト時 | 不正検知 / 監査 | `audit_logs.ip_address` |
| User-Agent | リクエスト時 | 障害解析 / セキュリティ | `audit_logs.user_agent` |
| 操作ログ（誰が何を何時に） | トリガ自動 | 監査 / 不正検知 | `audit_logs` |
| ログイン失敗履歴 | rate limit カウント | アカウントロック判定 | Vercel KV（最大 24 時間） |

### 1.4 取得しないもの（明示的非取得）

- 電話番号、住所、生年月日、性別、顔写真、健康情報
- 位置情報（GPS / ブラウザ Geolocation API）
- マイクロフォン / カメラ
- クレジットカード情報（直接入力させない、領収証画像中の末尾4桁は同意取得対象）
- マイナンバー / 法定識別子

## 2. 保存期間

| データ種別 | 保存期間 | 根拠 | 削除タイミング |
|-----------|---------|------|--------------|
| 領収証画像 / PDF | **7 年** | 法人税法第 59 条の 2 / 電帳法 Phase 2 移行に備える | 7 年経過後の年次バッチで物理削除 |
| 領収証メタデータ（金額・日付・店舗等） | **7 年** | 同上 | 同上 |
| `expenses` レコード | **7 年** | 経費精算の証憑として | 7 年経過後の年次バッチ |
| `approval_logs` | **7 年** | 承認証跡として領収証と同期間 | 同上 |
| `audit_logs` | **3 年** | 不正検知 / セキュリティ監査 | 3 年経過後の月次バッチ |
| 退会済みユーザーの個人情報（氏名・メール） | **退会から 30 日**（法的保存義務分は除く） | 個人情報保護法（保有最小化） | 30 日経過後の日次バッチ |
| パスワードハッシュ | 退会と同時に削除 | 個人情報保護法 | 退会フロー内 |
| ログイン失敗履歴（KV） | **最大 24 時間** | rate limit に必要な範囲 | KV TTL |
| Sentry エラーログ（Phase 1.5） | **30 日** | Sentry Free plan 既定 | Sentry 自動 |

**重要**: 領収証画像と `expenses` レコードは退会後も 7 年間保存する（経費証憑として法令上必要）。ただし `users.full_name` / `users.email` は退会 30 日後に NULL 置換し、`expenses` の関連表示は「退会済みユーザー」とマスキング表示する。

## 3. 退会時の削除フロー

### 3.1 フロー図

```
1. ユーザーが /settings/account から「退会する」ボタン押下
        │
        ▼
2. 確認ダイアログ + パスワード再入力（admin の場合は CEO 経由オーナー承認）
        │
        ▼
3. UPDATE users SET is_deleted=true, deleted_at=now()  ← 論理削除
        │
        ▼
4. ログアウト → 再ログイン不可（is_deleted=true でブロック）
        │
        ▼
5. 30 日間の異議申立期間
        │  （この間に「やっぱり戻したい」連絡があれば admin が論理削除を取り消し可）
        ▼
6. 日次バッチ（cron）で 30 日経過したユーザーを処理:
   a) UPDATE users SET email=NULL, full_name=NULL, hashed_password=NULL
   b) DELETE FROM auth_sessions WHERE user_id = ?
   c) DELETE FROM auth_accounts WHERE user_id = ?  ← Better Auth の OAuth 連携テーブル
   d) `audit_logs` の actor_id は NULL に置換せず保持（監査要件のため）
   e) `expenses` / `approval_logs` の user_id は保持（経費証憑として 7 年）
   f) `expense_attachments`（領収証画像）は保持（経費証憑として 7 年）
        │
        ▼
7. 監査ログに「ユーザー匿名化完了」エントリを INSERT
        │  diff = { before: { email: "***", full_name: "***" }, after: { email: null, ... } }
        │  ← 個人情報そのものは記録しない
        ▼
8. ユーザー画面表示時:
   - 退会済みユーザーの申請は「退会済みユーザー」と表示
   - 領収証中の氏名は引き続き映る（経費証憑のため改変不可）
```

### 3.2 admin の退会フロー特例

- admin（全体管理者）は単独で退会できない（運用継続性のため）。
- admin が退会する場合は、別の admin を立てるか、CEO 経由でオーナー承認を経て CEO/開発が DB 直接操作で対応。
- 上記手順を `RUNBOOK.md` に記載（【P1必須】）。

### 3.3 SaaS 化（Phase 2 以降）の追加考慮

- 組織オーナー（テニス部本部など）の退会 = テナント解約フローは Phase 2 で別途設計。
- テナント解約時は **30 日 + 60 日の二段階期間**（30 日異議 + 60 日 export 猶予）を設ける。

## 4. データエクスポート権（個人情報保護法 28 条 = 開示請求対応）

### 4.1 Phase 1 対応

- ユーザー自身の以下データを **メール窓口（CEO 経由）** で開示請求可能とする:
  - profile レコード
  - 自分が申請した expenses 全件
  - 紐づく領収証画像（zip）
  - 自分の audit_logs（30 日分）
- 開示請求は **2 週間以内** に対応（法定期間）。

### 4.2 Phase 2 対応（自動化）

- `/settings/data-export` 画面から **CSV + 領収証 zip** をワンクリックダウンロード可能にする。
- 実装: Server Action でユーザー自身の expenses を select → CSV 生成 → 領収証を Vercel Blob から zip → 一時 signed URL（TTL 1 時間）で配信。

## 5. 第三者提供 / 越境移転

### 5.1 第三者提供（個人情報保護法 27 条）

- **本サービスは自組織内利用のため、第三者提供は行わない**。
- 例外: 法令に基づく場合（捜査関係事項照会等）のみ。

### 5.2 越境移転（個人情報保護法 28 条）

- データは以下の **米国サーバー** で取り扱われる:
  - **Turso**（米国 Bay Area / 各国 edge region）: DB
  - **Vercel**（米国）: アプリ実行 / Blob ストレージ
  - **Cloudflare R2**（米国）: 領収証ストレージ（候補）
  - **Resend**（米国 / Delaware）: メール送信
  - **Vercel KV / Upstash**（米国）: rate limit
- 上記事業者の **十分性認定** または **GDPR/SCC 準拠** を確認しプラポリに明記。
- 利用者の同意取得: アカウント作成時に「米国サーバーでのデータ取扱に同意」のチェックボックスを必須化。

## 6. Cookie 利用

| Cookie 名 | 種類 | 必須 | TTL | 用途 |
|----------|------|------|-----|------|
| `coatly.session` | 認証 | ◯ | 30 日 | ログインセッション。SameSite=Strict; HttpOnly; Secure |
| `coatly.csrf` | 認証補助 | ◯ | セッション | CSRF トークン（Better Auth が発行） |
| `coatly.theme` | 設定 | × | 365 日 | ダーク/ライト切替（Phase 2 で実装） |

### 6.1 解析 Cookie（取得しない方針）

- **当面取得しない**。Vercel Analytics は **Cookie-less**（IP ハッシュベース）のため OK。
- Google Analytics 等の Cookie 系解析ツールは **Phase 2 以降に検討**、導入時はバナー表示 + opt-in 必須。

### 6.2 Cookie バナー

- 認証 Cookie のみのため、Phase 1 では Cookie バナーは **不要**（ePrivacy 指令でも認証 Cookie は同意不要）。
- Phase 2 で解析 Cookie を導入する際にバナー実装。

> ## プライバシーポリシー
>
> Coatly（以下「本サービス」）は、利用者の個人情報を、組織の活動費精算・予算管理を目的としてのみ取り扱います。
>
> 取得する情報は、メールアドレス・氏名・所属グループ・領収証画像・操作ログ等です。利用目的を超える利用や、第三者への提供は法令に基づく場合を除き行いません。

> データは米国の Vercel Inc.、Turso（ChiselStrike Inc.）、Resend Inc. のサーバーに保存され、いずれの事業者も適切な安全管理措置を講じています。領収証および経費データは法人税法に基づき 7 年間、操作監査ログは 3 年間保存し、その後削除します。

> 利用者は自身の個人情報の開示・訂正・利用停止・削除を、CEO 窓口（ai-lab@improver.jp）にメールで請求できます。退会時は氏名・メールアドレスを 30 日後に削除します（経費証憑は法令により 7 年保存）。
>
> 本ポリシーは個人情報保護法（改正）に準拠して運用し、変更がある場合は本サービス上で告知します。

## 10. 領収証画像中の個人情報の扱い（重要）

### 10.1 リスク

領収証画像には以下が映り込む可能性がある:
- 購入者氏名（手書き / 印字）
- 店舗住所・電話番号
- クレジットカード末尾 4 桁
- 購入時刻 + 場所 → 行動履歴推定
- 同行者情報（複数人分のレシートなら）

### 10.2 対策

| 対策 | 実装 | Phase |
|------|------|-------|
| **同意取得 UI** | 申請フォームに「領収証に氏名・カード末尾等が含まれる場合があり、その情報を含めて 7 年間保管することに同意する」チェックボックスを必須化 | 【P1必須】 |
| **アクセス制御** | 領収証は private bucket + signed URL TTL=60s + アプリ層認可（自分 / 同グループ管理者 / admin のみ閲覧可） | 【P1必須】 |
| **マスキング機能** | カード末尾 4 桁等を画像処理で塗りつぶす機能 | 【P3余裕で】（OCR 連携時） |
| **削除権** | 申請の差戻 / 取消時に領収証も削除 | 【P1必須】 |
| **ログ保存** | 誰がいつどの領収証を閲覧したかを `audit_logs` に記録 | 【P1必須】 |
